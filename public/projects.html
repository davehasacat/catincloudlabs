Enterprise Stock Market ELT (reference implementation)

A production-grade pipeline built on AWS, MWAA, Snowflake, and dbt, designed to ingest and transform stock and options data in a private VPC. It uses KMS-encrypted S3 buckets, VPC endpoints, and incremental dbt models to keep data flowing reliably from raw JSON to analytics-ready marts.

This is the same architecture pattern I adapt for client environments that want calm, well-understood data platforms.

At a glance

  • Stack: AWS (VPC, S3, MWAA), Snowflake, dbt
  • Pattern: private networking, KMS encryption, incremental models
  • Built for: calm, production-like behavior

Architecture and security posture

The high-level architecture and security/observability diagrams below show how the pieces fit together: MWAA in a private VPC, KMS-encrypted S3 buckets, Snowflake, and dbt all wired through VPC endpoints and least-privilege access.

Private MWAA Environment

A hardened Airflow deployment running on AWS MWAA inside a fully private VPC. MWAA connects to ECR, CloudWatch Logs, KMS, Secrets Manager, and S3 through VPC endpoints.

In progress: Dashboards & apps on top

A browser-based analytics layer built with tools like Plotly Dash on top of Snowflake marts. The goal is to provide straightforward views into pipeline health and market data without adding another brittle surface area to maintain.

Future: Machine learning on top of calm pipelines

The same stock + options pipeline that powers these charts is already keeping months of joined history in clean Snowflake marts.
