Enterprise Stock Market ELT (reference implementation)

A production-grade pipeline built on AWS, MWAA, Snowflake, and dbt, designed to ingest and transform stock and options data in a private VPC. It uses KMS-encrypted S3 buckets, VPC endpoints, and incremental dbt models to keep data flowing reliably from raw JSON to analytics-ready marts.

This is the same architecture pattern I adapt for client environments that want calm, well-understood data platforms.

At a glance

  • Stack: AWS (VPC, S3, MWAA), Snowflake, dbt
  • Pattern: private networking, KMS encryption, incremental models
  • Built for: calm, production-like behavior rather than a one-off demo

All three views below come from the same Snowflake marts that power this reference pipeline. For client work, those marts would typically sit behind a BI tool or custom app; here they're exposed as static JSON slices for the demo.

Daily price & options volume for five large-cap tickers (Sep 2 - Nov 28, 2025)

Daily features snapshot: price, returns, and options activity

Options chain: top contracts over a recent window

This table surfaces the option contracts that carried the most flow over a recent trading window, across five large-cap tickers. Each row rolls up total volume for a single contract across that window, with a snapshot of where it sits on the chain. The table is powered by data exported from the same Snowflake marts into a static JSON slice for this demo.

Columns show expiry, option type, strike, latest close price observed in the window, total volume across the window, days to expiration at the end of the window, and average moneyness versus the underlying price. You can tap or click the column headers to re-sort (for example, by total volume, expiry, or moneyness).

Private MWAA Environment

A hardened Airflow deployment running on AWS MWAA inside a fully private VPC. MWAA connects to ECR, CloudWatch Logs, KMS, Secrets Manager, and S3 through VPC endpoints. This ensures orchestration, logging, and image pulls all happen without public internet access.

The same approach can be used to keep your schedulers and control plane traffic private while still integrating cleanly with AWS services and your data warehouse.

In progress: Dashboards & apps on top

A browser-based analytics layer built with tools like Plotly Dash on top of Snowflake marts. The goal is to provide straightforward views into pipeline health and market data without adding another brittle surface area to maintain.

As the core platform stabilizes, these kinds of dashboards and lightweight apps become much easier to add safely.
